# Mayday для Windows

[English version](README.md)

Mayday для Windows — Flutter-приложение для управления защищенным подключением Mayday в обычном Windows-интерфейсе. Оно рассчитано на повседневное использование: импортировать ключ доступа, подключиться или отключиться, свернуть приложение в системный трей и быстро вернуться к нему при необходимости.

Этот README описывает приложение на пользовательском уровне и намеренно не затрагивает внутренние детали подключения.

## Возможности

- Подключение и отключение одной основной кнопкой.
- Интеграция с системным треем Windows.
- Разные иконки в трее для включенного и выключенного состояния с учетом светлой или темной темы Windows.
- Закрытие окна в трей, пункты **Show Mayday** и **Exit** в меню трея.
- Поведение одного экземпляра: повторный запуск открывает уже существующее окно.
- Импорт только через ключ доступа.
- ID пользователя/конфигурации отображается из импортированного профиля и не редактируется вручную.
- Локальные настройки приложения.
- Поддержка русского и английского языка интерфейса.
- Опциональный автозапуск Mayday после входа в Windows.
- Выбор Windows-приложений для раздельной маршрутизации.
- Фоновая проверка рискованных приложений с сохранением результатов и диалогом перед подключением.
- Расширенные настройки: режим транспорта, network rescue, MTU, размер packet_fragment, packet padding, batching пакетов и опциональные метрики.
- Проверка обновлений через GitHub Releases с небольшим баннером.
- Экран диагностики с понятной проверкой готовности.
- Проверка прав администратора и сценарий перезапуска с нужными правами.

## Как Используется

Откройте Mayday, импортируйте ключ доступа, выданный для вашей учетной записи, и нажмите **Подключиться**. Главный экран показывает текущее состояние и ID пользователя/конфигурации из импортированного профиля. Технические детали находятся в раскрываемых разделах, чтобы обычный экран оставался простым.

Mayday проверяет установленные приложения, exe-файлы, службы и автозадачи из списка риска в фоне при новой сессии приложения. Если проверка не была успешно пройдена, при подключении открывается диалог с объяснением риска и двумя действиями: принять возможные последствия и подключиться либо запустить проверку. Результаты проверки можно открыть повторно; полный путь найденного объекта можно выделить или скопировать.

Когда подключение активно, иконка в трее меняется на включенный вариант. При закрытии окна Mayday остается в трее. Через меню трея можно снова показать окно или выйти из приложения. При выходе через трей Flutter-слой сначала останавливает активное подключение, а затем нативный Windows runner закрывает приложение.

Если сохраненный профиль создан для старого контракта runtime, Mayday попросит импортировать свежий ключ доступа и не будет использовать старый профиль молча.

## Настройки

Раздел настроек содержит только пользовательские элементы управления:

- Поведение приложения при входе в Windows.
- Данные импортированного профиля, включая ID пользователя/конфигурации только для чтения.
- Режим транспорта, расширенные сетевые параметры защиты, включая packet padding, и опциональные метрики.
- Выбор Windows-приложений для раздельной маршрутизации.
- Диагностика и состояние готовности.
- Выбор языка интерфейса.

Большинству пользователей достаточно импортировать ключ доступа, пользоваться основной кнопкой подключения и меню в трее.

## Flutter-Слой

Mayday организован как небольшое слоистое Flutter desktop-приложение:

```text
lib/
  main.dart                 точка входа приложения
  app/                      тема, язык, верхнеуровневая связка
  core/
    models/                 модели данных для интерфейса
    services/               локальное хранение, платформенные мосты, пикеры
    l10n/                   встроенные русские и английские тексты
  features/home/
    application/            координация экрана
    presentation/           view model, страницы и виджеты
windows/                    нативный Windows runner и интеграция с треем
```

Flutter-слой отвечает за интерфейс, состояние экранов, локализацию, импорт, локальные настройки приложения и управление подключением. Нативный Windows runner отвечает за desktop-интеграцию: трей, активацию уже открытого окна и жизненный цикл окна.

## Приватность И Локальные Данные

Mayday хранит данные приложения локально для текущего пользователя Windows. Чувствительные пользовательские данные защищаются на диске средствами Windows там, где это применимо. README не описывает внутренние форматы данных и устройство подключения.

## Если Что-То Не Работает

Если Mayday не подключается:

- Убедитесь, что приложение запущено с теми правами, которые оно запросило.
- Откройте уже запущенное окно Mayday из трея или меню Пуск.
- Импортируйте ключ доступа заново, если приложение сообщает, что сохраненный профиль несовместим или настройка не завершена.
- Запустите проверку приложений и посмотрите диалог результатов, если Mayday предупреждает перед подключением.

Автозапуск открывает приложение после входа в Windows. Он не включает подключение автоматически.
